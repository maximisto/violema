# Agent Studio Repo Legal Files

## Purpose

These are the legal and governance files the standalone public repository should ship with on day one.

## Required files

- `LICENSE`
- `TRADEMARKS.md`
- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- `SECURITY.md`

## License recommendation

Use:

- MIT

Reason:

- lowest-friction adoption
- strong compatibility with open-source developer tooling
- best fit for an adapter ecosystem

## Trademark policy recommendation

Add `TRADEMARKS.md`.

This file should make clear:

- the code is MIT licensed
- the names `Purple Orange`, `Purple Orange AI`, and `Agent Studio` are reserved trademarks or product marks if we choose to enforce them
- forks can use the code
- forks should not imply official affiliation through branding or logos

## Suggested file contents

### `LICENSE`

Use the standard MIT license text.

### `TRADEMARKS.md`

Suggested structure:

- what is covered
- what the license allows
- what the license does not grant
- allowed nominative use
- prohibited confusing branding use
- contact for permission

### `CONTRIBUTING.md`

Suggested sections:

- how to set up locally
- how to run the demo
- how to add an adapter
- test expectations
- PR guidelines

### `CODE_OF_CONDUCT.md`

Use a standard contributor covenant style document or a short direct policy if we want something simpler.

### `SECURITY.md`

Suggested sections:

- supported versions
- how to report security issues
- expected response path
- disclosure policy

## Recommendation

Do not over-lawyer the repo.

Keep:

- MIT simple
- trademark file clear
- contribution process light

That is enough to launch credibly.
