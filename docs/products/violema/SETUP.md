# Violema Setup

## One-line definition

Violema is the AI operator for recurring team workflows.

## Product job

Violema should help a team hand off real work, let it run, and trust the result.

The core promise is:

- connect tools
- define a workflow
- run it reliably
- get useful output
- repeat with confidence

## Who it is for

Primary user:

- founder
- operator
- product lead
- GTM lead
- finance or research-heavy solo operator

Best fit:

- small teams with repeated operational work
- people who want output, not architecture theory
- users already living in Slack, email, docs, and dashboards

Worst fit:

- hobbyist tinkerers looking for a sandbox
- users who want to design agent topologies first
- people who mainly want a coding-agent lab

## What it is not

Violema is not:

- a generic chatbot
- a visual agent builder
- an infrastructure tuning lab
- an “AI company simulator”

Those ideas belong either in the background or in Agent Studio.

## Product thesis

The winning Violema experience is simple:

- one clear workflow gets set up fast
- it runs reliably
- the result is useful
- the next workflow is obvious

The user should feel:

- less manual work
- less follow-up debt
- more trust over time

## Core user loop

1. Connect the tools that matter.
2. Create one high-value recurring workflow.
3. Let Violema run it.
4. Review the result quickly.
5. Approve, adjust, or scale usage.

## Initial wedge

Start with recurring work for startup teams:

- research and monitoring
- briefings and summaries
- internal updates
- recurring reporting
- follow-through workflows with approvals

Do not lead with “do everything.”

## Product surfaces

Keep these as first-class:

- dashboard
- workflows
- runs
- integrations
- billing
- settings

Keep `Agent Studio` secondary or separate.

## UX principles

- Outcome first.
- Architecture mostly invisible.
- Reliability over cleverness.
- One obvious next action on every screen.
- Advanced controls only when earned.

## Packaging and monetization

Sell:

- completed work
- workflow capacity
- higher autonomy
- more integrations
- more run budget

Do not sell:

- raw token logic
- architecture jargon
- abstract “AI power”

## 30 / 60 / 90 day focus

### 30 days

- stabilize workflow creation and editing
- make run history dependable
- simplify the dashboard around outcomes
- tighten the product story on the landing page

### 60 days

- make one workflow category clearly excellent
- improve approvals, alerts, and delivery surfaces
- make onboarding faster and more opinionated

### 90 days

- deepen recurring workflow templates
- improve team-facing surfaces
- add stronger retention through reliable daily value

## Success metrics

- workflows created per activated workspace
- first successful run rate
- repeat run rate
- weekly active workspaces
- approval completion rate
- output acceptance rate

## Relationship to Agent Studio

Violema should integrate with Agent Studio, not depend on it.

Agent Studio can provide:

- policy suggestions
- experiment winners
- promotion and rollback recommendations

But Violema should still make sense without opening Agent Studio once.
